Article HTML layout #

The main body of the article should be enclosed by an element with article-body class. The article-body should be enclosed by an element with article-container class. This allows styles to be scoped to the actual contents and for sidenote and aside placement/sizing to work.

Text components #

Headings #

Headings are useful for semantically organizing sections of a document. You should use them in favor of large or bold text that carries no semantic meaning. Headings have levels that indicate a parent–child relationship between sections. Except heading level 1 (the title), a heading of level n should be under by a heading of level n − 1; do not skip levels. To avoid confusion, you should avoid using heading levels greater than 3 or 4.Practical Typography: Headings

Links #

Links are distinguished by underline for accessibility.

A link to another website that is annotated with the link-external class or has a URL beginning with http://, https://, or mailto: will have a different indicator.

Asides and sidenotes #

Asides and sidenotes both appear in the right margin of the page. However, they are different!

This is an aside. It uses the HTML aside element.

Asides can contain block content, such as figures, lists, and entire paragraphs.

Asides can contain block content and are placed next to block-level elements (i.e., you cannot align an aside with a specific line in a paragraph). In the event that the screen size is too small, they move into the main content flow, above the content they are ordinarily placed next to.

In contrast, sidenotes can only contain inline content and are placed inline with the text they refer to.This is an example of a sidenote. Notice how, with a wide enough screen, it’s vertically aligned with its number. They are a replacement for footnotes, allowing readers to access tangential content without scrolling to the bottom of the page. In the margins, they are aligned with the line that they occur in (as long as another element does not push it down). They can optionally be numbered. In the event that the screen size is too small, sidenotes disappear and must be expanded by clicking on their corresponding indicator/number.

Don’t put sidenotes or asides inside of each other. Don’t put sidenotes in tables; use asides or another form of annotation instead.

Lists #

List items can contain block elements. If they do, enclose all text with p tags for consistent loose spacing. If all list items are inline, then you can omit p around text for compact spacing.
